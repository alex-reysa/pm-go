# Spec Input Template

Use this structure for the primary user-provided spec document.

## Title

One clear sentence describing the desired feature or change.

## Objective

Describe the business or product outcome.

## Scope

- In scope items
- Systems or packages expected to change
- External dependencies or integrations

## Out of Scope

- Explicit exclusions

## Constraints

- Risk or safety constraints
- Performance or latency limits
- Security considerations
- Migration constraints

## Acceptance Criteria

- Concrete user-visible behaviors
- Required tests or validation commands
- Definition of done

## Repo Hints

- Suspected files or packages involved
- Known commands
- Known related changes

## Open Questions

- Items that require planning audit or human clarification
