## Operator interventions

This run took 5 interventions. The AXON 2026-04-28/29 run took 7+;
slice 0 alone is not the reason for the reduction (that was a smaller
spec). The interventions are catalogued for v0.8.8.x scoping.

1. **DB hand-edit to reset phase from `blocked` → `executing` after
   merge-validation false-fail.** The integration validate step ran
   `pnpm test` against the merged tree and one test in `run.test.ts`
   asserted `process.env.PLANNER_RUNTIME === 'sdk'` while the worker
   process had `PLANNER_RUNTIME=auto` exported, leaking through to the
   subprocess. Pre-existing test hermeticity bug surfaced by the
   integration env. **Fixed in-band:** the slice-1 implementer for the
   readiness-probe task (`104827e`) added `delete process.env[k]`
   setup/restore to the test. This bug should now stay closed; future
   integration runs should not need `PM_GO_INTEGRATION_SKIP_VALIDATE=1`
   on this repo.
2. **Cross-task contract drift caught by the phase auditor (not the
   per-task reviewer).** Task 1 (CLI helper) shipped
   `service === 'pm-go'`; task 2 (API) shipped `service: 'pm-go-api'`.
   Per-task reviews approved both in isolation. The phase auditor
   composed the merged head and immediately flagged the drift with a
   high-confidence finding. **Fix path is the gap:** there is no
   public endpoint to send a `ready_to_merge` task back into the fix
   loop, so the operator hand-edited the agent worktree, committed a
   fixup, and re-triggered integrate. Slice 2's `coverageHints` would
   catch this earlier (pre-review) by listing `service: 'pm-go-api'`
   as a `mustTouchSymbol` on the helper task.
3. **DB hand-edit to clear stale phase audit + merge_run + lease**
   between the failed audit and the re-trigger. Same shape as (1).
4. **Workflow termination on the `runCompletionAuditor` crash loop.**
   The SDK reports
   `Claude Code native binary not found at .../claude` while the
   binary at that path exists, is executable, and reports
   `2.1.114 (Claude Code)` when invoked directly. Other roles
   (planner, implementer, reviewer, phase-auditor) using the same SDK
   path completed cleanly. Out of scope for v0.8.8.0; matches the
   AXON evidence as a reliable repro on this machine.
5. **Source-tree state restore from HEAD.** After `pnpm -r build`
   finished post-merge, `git status` showed every slice-0 file
   staged-as-reverted (HEAD had the new content but the index +
   working tree had the pre-merge content). Restored via
   `git restore --staged --worktree .`. Root cause unknown — worth
   investigating whether a pnpm `prepare` hook, a leftover integration
   activity, or the supervisor's own shutdown path mutates the host
   repo's index.

## Bugs to file (input for v0.8.8.x specs)

- **Phase auditor finding routes back to a re-fix loop.** A finding
  like the contract drift above currently dead-ends at a hand-fixup.
  A new endpoint (e.g. `POST /tasks/:id/reopen`) that re-opens a
  `ready_to_merge` task with a populated finding payload would close
  the autonomy gap. Adjacent to slice 2's coverage-hints work but
  worth its own AC.
- **`runCompletionAuditor` SDK path mismatch.** Reproduce in
  isolation. Likely a difference in how the completion-auditor
  invokes `query()` vs the other auditors — perhaps a working
  directory, env var, or sub-process spawn flag. Tracked separately
  per the parent spec (`docs/specs/v0.8.8-completion-auditor-repro.md`,
  to file).
- **`pm-go ps` state-file vs live-process drift.** Several times
  during the run, `pm-go ps` reported `(no pm-go-owned processes
  recorded)` while `pgrep` showed the supervisor + api + worker
  alive. Reverse case also occurs (state file lists dead PIDs).
  `pm-go recover` should be the canonical reconciliation path; the
  test cases that exercise it should include "live process not in
  state" and "state lists dead PID" both.
- **Mystery source-tree revert during/after `pnpm -r build`.** See
  intervention (5). Repro: FF main to a multi-commit integration head,
  run `pnpm -r build`, then `git status`. If staged reverts appear,
  there's an automation somewhere; otherwise this was a one-off.

## What to keep

- The phase auditor caught a real cross-task bug per-task review
  missed. **Phase auditing is doing its job.** Slice 2 should
  augment, not replace, the phase audit.
- The v0.8.7.1 supervisor fail-open held when `runCompletionAuditor`
  crashed: stack stayed up, plan stayed in DB, operator could
  diagnose. Worth keeping in mind when designing slice 0+1+2 error
  paths.
- The integration `pnpm install` + `pnpm -r build` step is slow but
  honest — it caught the test hermeticity bug that pre-existed in
  main. Don't make `PM_GO_INTEGRATION_SKIP_VALIDATE=1` the default.
